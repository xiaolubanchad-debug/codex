# UI 设计说明

## 1. 参考图提炼

参考图的可复用视觉语言主要有以下几个特征：

- 顶部导航简洁，品牌标识突出
- 首屏采用大图背景 + 居中的白色信息卡片
- 内容区大量使用三列卡片布局
- 栏目标题前后有明显的红色强调线
- 底部有强烈品牌色订阅区与深色页脚

本项目不会直接复刻该页面，而是保留其“编辑型内容站”的气质，并重构为更现代、适配 `Next.js` 博客体验的版本。

## 2. 全站视觉语言

### 色彩

- 主色：`#d92c1f`
- 深色：`#121212`
- 纸张白：`#fbf8f4`
- 辅助米灰：`#efe7dc`
- 文本黑：`#171717`
- 次级文本：`#6a625b`

### 字体建议

- 标题：偏编辑感的衬线或准衬线字体
- 正文：高可读无衬线字体
- 数字/标签：更紧凑，强调信息密度

### 组件气质

- 卡片边角不宜过圆
- 阴影克制，强调“纸片叠放”感
- 图片区域可用渐变和抽象视觉代替真实封面
- 强调块和按钮尽量使用红色但避免整页过载

## 3. 页面设计说明

### 首页

目标：

- 建立品牌记忆
- 展示栏目结构
- 强化精选内容

结构：

1. 顶部导航
2. Hero 主视觉
3. 精选文章
4. 分栏目内容区
5. 订阅区
6. 页脚

设计重点：

- Hero 用大面积深色渐变和几何装饰营造内容品牌感
- Hero 白色卡片承接主标题、副标题、关键数据
- 栏目内容区采用三列卡片，保持截图中的“杂志墙”观感

### 列表页

目标：

- 让用户专注于“某个栏目下有哪些内容”

结构：

1. 栏目头图/简介
2. 过滤标签
3. 文章网格
4. 翻页或查看更多按钮

设计重点：

- 头部保留栏目辨识度
- 卡片网格统一但允许精选卡片跨列，提高节奏感

### 详情页

目标：

- 长文阅读舒适
- 强化文章信息层级
- 支持继续浏览

结构：

1. 文章头图 / 标题区
2. 元信息
3. 正文
4. 关键引用块
5. 相关推荐

设计重点：

- 标题区需要强层级和留白
- 正文宽度控制在舒适阅读范围
- 引用、分隔、标签的视觉语言需要统一

### 搜索页

目标：

- 快速检索内容
- 搜索过程清晰、反馈直接

结构：

1. 搜索标题区
2. 大搜索框
3. 热门关键词
4. 搜索结果区
5. 空状态

设计重点：

- 搜索框要成为视觉主角
- 结果区卡片信息要比首页更密集

### 关于页

目标：

- 解释站点是谁、为什么存在、写什么

结构：

1. 品牌介绍
2. 内容原则
3. 栏目说明
4. 时间线或工作方式
5. 联系方式

设计重点：

- 需要保留品牌气质，不做成纯文字说明书
- 可以用数字、分栏、引用等方式丰富内容表达

## 4. 交互原则

- Hover 反馈轻量但明确
- 卡片点击区域应尽量完整
- 搜索结果应即时响应
- 移动端优先保证层级、点击热区和可读性

## 5. 响应式建议

- Desktop：三列主布局
- Tablet：两列
- Mobile：单列，Hero 卡片改为顺序堆叠

## 6. 实现建议

- 使用共享卡片组件统一样式
- 通过分类、标签、精选状态控制卡片变体
- 颜色、间距、阴影用 CSS 变量统一管理
