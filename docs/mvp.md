# 博客站点 MVP 文档

## 1. 项目定位

打造一个以内容阅读为核心的博客站点，视觉方向参考提供的截图：

- 白底内容区 + 红色强调色
- 大图头部 + 浮层信息卡片
- 杂志感内容编排
- 明确的栏目分区
- 强调“编辑精选”和“持续更新”的内容氛围

站点应兼顾品牌展示、内容沉淀、文章检索与作者介绍，首版采用 `Next.js + Node` 技术栈实现。

## 2. MVP 目标

首版只解决三件核心事情：

1. 用户可以快速感知站点定位与内容调性。
2. 用户可以高效浏览文章列表并进入详情阅读。
3. 用户可以通过搜索和关于页理解作者/团队与内容方向。

## 3. 目标用户

- 对科技、互联网、产品、游戏、创作类内容感兴趣的读者
- 希望快速浏览精选内容并继续深读的回访用户
- 通过搜索寻找特定话题文章的目标用户
- 想了解站点背景、作者理念、内容范围的初次访问者

## 4. MVP 页面范围

本次 MVP 包含 5 个页面：

1. 首页
2. 列表页
3. 详情页
4. 搜索页
5. 关于页

## 5. MVP 核心功能

### 首页

- 展示站点品牌、核心导航、精选文章
- 首屏 Hero 区强化品牌和当期主打内容
- 按栏目分组展示文章卡片
- 提供 Newsletter/订阅入口

### 列表页

- 以网格或瀑布感的编排展示某栏目全部文章
- 支持按栏目查看
- 展示文章摘要、发布时间、标签、阅读时长

### 详情页

- 完整展示文章内容
- 展示标题、摘要、作者、发布时间、标签
- 提供相关推荐，形成内容回流

### 搜索页

- 提供关键词搜索
- 实时筛选文章标题、摘要、标签
- 展示搜索结果数量和空状态

### 关于页

- 介绍站点使命、内容方向、作者/团队
- 说明内容栏目与更新方式
- 提供联系或合作入口

## 6. MVP 不包含

- 用户登录/注册
- 评论系统
- 后台 CMS
- 点赞/收藏/分享统计
- 多作者权限管理
- 国际化
- 复杂推荐算法

## 7. 内容模型

单篇文章至少包含：

- `title`
- `slug`
- `category`
- `excerpt`
- `publishedAt`
- `readingTime`
- `author`
- `tags`
- `cover` 或视觉主题信息
- `body`
- `featured`

## 8. 页面成功标准

### 首页

- 用户 5 秒内能理解站点主题和主要栏目
- 用户能明显看到主打文章与最新内容

### 列表页

- 用户可以快速扫读标题和摘要
- 用户能在 1 次点击内进入详情页

### 详情页

- 内容阅读层级清晰
- 相关推荐能支持继续阅读

### 搜索页

- 输入关键词后结果变化清晰
- 无结果时有明确反馈和返回路径

### 关于页

- 用户能理解站点做什么、由谁运营、适合谁阅读

## 9. 视觉方向

- 整体基调：现代杂志感、内容型、克制但有辨识度
- 主色：品牌红
- 背景：暖白 / 浅灰，搭配暗色大图区
- 卡片：留白明显，适度阴影，强调层级
- 字体策略：标题偏个性，正文偏清晰耐读

## 10. 首版技术建议

- `Next.js App Router`
- 静态内容或本地 Mock 数据先行
- 后续再接入 Markdown / Headless CMS / 数据库
- 搜索页 MVP 阶段使用前端过滤
